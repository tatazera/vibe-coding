Contrato de dados (payload JSON esperado)
O n8n deve enviar um array de objetos nesse formato para o endpoint — ou chamar window.S1.receberEntrada(obj) diretamente.
[ { "id": "uuid-único", // obrigatório "who": "Francine", // remetente no WhatsApp "solic": "Francine", // solicitante do projeto "nome": "CLIENTE EVENTO 2026", // nome do projeto "prior": "Alta|Média|Nula", // prioridade extraída "obs": "Júnior", // responsável pelo desenho "prazo": "2026-08-15", // YYYY-MM-DD (opcional) "nota": "texto livre", // observações (opcional) "msg": "mensagem original", // texto bruto do WhatsApp "time": "11/06/2026, 14:30" // data/hora da mensagem } ]
Simular entrada WhatsApp
Cole um JSON válido (objeto ou array) para testar o fluxo sem precisar do WhatsApp conectado.
Como conectar
Opção A — n8n + Evolution API (recomendado)
1. Configure a Evolution API para escutar o grupo do WhatsApp
2. No n8n: Webhook → Filtrar mensagens → Claude API (extrair dados) → HTTP Request (POST para este dashboard ou escrever em JSON)
3. Configure a URL acima para o n8n fazer polling ou use window.S1.receberEntrada() via script

Opção B — postMessage (iframe/Electron)
Envie window.postMessage({type:'s1_entrada', data: [...]}, '*') de qualquer página pai ou wrapper
